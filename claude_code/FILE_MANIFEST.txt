PROJECT OVERVIEW:
A complete Retrieval-Augmented Generation (RAG) chatbot for ISO 13485:2016
Medical Device Quality Management Systems built with Streamlit and OpenAI.

================================================================================
EXECUTABLE FILES (Run These)
================================================================================

1. iso_13485_rag.py
   - Type: Main Streamlit Web Application
   - Purpose: Professional web interface for RAG chatbot
   - How to run: streamlit run iso_13485_rag.py
   - Opens in: Web browser (http://localhost:8501)
   - Features:
     * 4 navigation modes (Search, RAG Assistant, Browse, About)
     * Professional UI with custom CSS styling
     * Keyword and ID-based procedure search
     * OpenAI RAG integration for intelligent answers
     * Detailed procedure viewing
     * Responsive design
   - Dependencies: streamlit, openai, python-dotenv
   - Time to run: <2 seconds startup

2. test_rag_demo.py
   - Type: Interactive Command-Line Demo
   - Purpose: Test RAG functionality without browser
   - How to run: python test_rag_demo.py
   - Features:
     * Interactive command-line interface
     * 2 modes: Interactive or Demo
     * Full RAG functionality
     * No browser required
     * Perfect for testing
   - Commands: search, find, ask, browse, section, examples, help, exit
   - Dependencies: openai, python-dotenv

================================================================================
SUPPORTED SEARCH METHODS
================================================================================

1. KEYWORD SEARCH
   Input: Any relevant word (e.g., "complaint")
   Returns: All procedures matching the keyword
   Speed: <1 second
   Best for: Exploring related procedures

2. PROCEDURE ID SEARCH
   Input: Exact procedure ID (e.g., "PROC_8_2_2")
   Returns: Specific procedure with full details
   Speed: <0.5 seconds
   Best for: Direct access to known procedures

3. RAG ASSISTANT QUERY
   Input: Natural language question (e.g., "How to handle complaints?")
   Returns: AI-generated answer with procedure references
   Speed: 3-5 seconds
   Best for: Getting guidance and implementation help

4. BROWSE ALL
   Input: None required
   Returns: All 45 procedures with filtering/sorting
   Speed: <1 second
   Best for: Exploring full system and understanding structure

TROUBLESHOOTING REFERENCE
================================================================================

Issue: "OPENAI_API_KEY not found"
Solution: Create .env file with your API key

Issue: "No module named streamlit"
Solution: Run pip install -r requirements.txt

Issue: "No procedures found"
Solution: Try different keywords or use browse mode

Issue: "OpenAI API Error"
Solution: Verify API key is valid and has credits

Issue: Slow responses
Solution: Normal (3-5 seconds for RAG), check internet

Issue: Port 8501 already in use
Solution: Streamlit will use next available port

Issue: Cannot connect to OpenAI
Solution: Check internet connection, verify API key
